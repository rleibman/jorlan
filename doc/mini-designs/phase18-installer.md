# Phase 18 Mini-Design: Installer and Distribution

## Goal

Jorlan can be installed cleanly on Ubuntu/Debian (two .deb packages) and macOS (Homebrew) by a
non-developer, with CI/CD packaging on every build and GitHub Release artifacts on version tags.

---

## Decisions

### Two Separate .deb Packages

| Package | Contents | System user | systemd service |
|---|---|---|---|
| `jorlan-server` | server JARs, systemd unit, init-db.sh, env template | `jorlan` | yes |
| `jorlan-shell` | shell JARs, launch script | â€” | no |

Separate packages let users install the shell on a remote machine pointing at a server elsewhere.

### Java Requirement

Both packages declare `default-jre-headless (>= 2:1.21)` as an `apt` dependency.
No JRE is bundled. On macOS the Homebrew formula declares `depends_on "openjdk@21"`.

### macOS Primary Distribution: Homebrew Tap

Homebrew is the easiest installation path for technical macOS users.

- User flow: `brew tap rleibman/jorlan && brew install jorlan && brew services start jorlan`
- The formula's `service` block instructs `brew services` to manage the LaunchDaemon plist.
- A separate GitHub repository `rleibman/homebrew-jorlan` holds the tap. For now the formula
  stub lives in `Formula/jorlan.rb` in the main repo; it is copied to the tap repo on release.
- `install-macos.sh` is provided as a fallback for non-Homebrew users; it extracts the tarball
  and loads the bundled LaunchDaemon plist via `launchctl`.

## Package Layouts

### jorlan-server.deb

```
/usr/lib/jorlan-server/          JARs + classpath (managed by sbt-native-packager)
/usr/bin/jorlan-server           launch script (symlink from above)
/etc/jorlan/server.env           env variables file (.withConfig() â€” not overwritten on upgrade)
/etc/jorlan-server/logback.xml   logging config (.withConfig())
/var/log/jorlan-server/          log directory (owned jorlan:jorlan, created in postinst)
/usr/lib/jorlan-server/scripts/init-db.sh   DB bootstrap script
/lib/systemd/system/jorlan-server.service   generated by SystemdPlugin
```

### jorlan-shell.deb

```
/usr/lib/jorlan-shell/           JARs + classpath
/usr/bin/jorlan                  launch script (executableScriptName = "jorlan")
```

## macOS Distribution

### Universal tarball (both packages)

`sbt server/universal:packageZipTarball` and `sbt shell/universal:packageZipTarball` produce
platform-agnostic tarballs. These are attached to GitHub Releases for macOS installation.

The server tarball includes:
- `bin/jorlan-server`
- `lib/*.jar`
- `conf/application.conf` (template)
- `conf/logback.xml` (template)
- `scripts/init-db.sh`
- `conf/server.env.example`

### Homebrew Formula (`Formula/jorlan.rb`)

- References the server tarball from a GitHub Release URL (TODO placeholder until first release).
- `depends_on "openjdk@21"`.
- `service` block lets `brew services start jorlan` manage the LaunchDaemon automatically.
- `caveats` section walks the user through init-db and env file setup.
- A corresponding `Formula/jorlan-shell.rb` handles the shell.

### Fallback: `install-macos.sh`

For users without Homebrew. Extracts the server tarball to `/opt/jorlan-server`, copies the
bundled LaunchDaemon plist to `/Library/LaunchDaemons/io.jorlan.server.plist`, and loads it
via `launchctl`. Requires `sudo`.

### LaunchDaemon plist (`io.jorlan.server.plist`)

Included in the server tarball under `launchd/`. Key settings:
- `Label`: `io.jorlan.server`
- `ProgramArguments`: `["/opt/jorlan-server/bin/jorlan-server"]`
- `EnvironmentVariables`: reads from `/etc/jorlan/server.env` (loaded by install script)
- `KeepAlive`: true
- `StandardOutPath` / `StandardErrorPath`: `/var/log/jorlan-server/server.log`
- `UserName`: `jorlan` (the install script creates this system user)

## CI/CD

### Build job (on every push/PR to main)

Added to `.github/workflows/scala.yml`:
- `sbt --error "server/debian:packageBin; shell/debian:packageBin"` after tests pass.
- Uploads jorlan-server_*.deb and jorlan-shell_*.deb as workflow artifacts.

### Release workflow (`.github/workflows/release.yml`)

Triggered on `v*` tags:
1. Run tests.
2. Build jorlan-server.deb and jorlan-shell.deb.
3. Build server macOS tarball (`server/universal:packageZipTarball`).
4. Build shell macOS tarball (`shell/universal:packageZipTarball`).
5. Attach all four artifacts to the GitHub Release.
